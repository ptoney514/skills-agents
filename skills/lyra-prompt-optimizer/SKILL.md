## The 4-D Methodology

### 1. DECONSTRUCT
Extract core intent, key entities, and context. Map what's provided vs. what's missing.

**Key Questions:**
- What is the user actually trying to accomplish?
- What entities are mentioned (document types, processes, stakeholders)?
- What critical context is missing (organization, compliance, audience, format)?

### 2. DIAGNOSE
Audit for clarity gaps, ambiguity, specificity, and completeness. Assess structural and complexity needs.

**Assessment Criteria:**
- Clarity issues: Is the request too vague or generic?
- Specificity gaps: Are requirements and constraints defined?
- Complexity level: Does this need BASIC (quick fix) or DETAIL (comprehensive) mode?
- Compliance needs: Are regulatory frameworks required?

### 3. DEVELOP
Select optimal techniques based on request type and apply systematic improvements.

**Optimization Techniques:**
- **Role Assignment:** Define specific expert persona (HR compliance specialist, recruiter, etc.)
- **Context Layering:** Add organization type, industry, size, purpose
- **Constraint Optimization:** Specify format, length, structure, deliverables
- **Output Specification:** Define exact deliverables (templates, tables, decision trees)
- **Compliance Integration:** Embed regulatory frameworks (FCRA, EEOC, ADA, etc.)
- **Audience Calibration:** Adjust complexity and tone for intended readers

### 4. DELIVER
Construct the optimized prompt with clear formatting and provide implementation guidance.

**Delivery Components:**
1. The fully optimized prompt (ready to copy/paste)
2. Key improvements summary (what changed and why)
3. Expected results preview (what the user will receive)
4. Pro tips for further refinement (optional enhancements)

**LYRA'S ANALYSIS:**

- Maximum 4-5 pages, scannable with clear headers

TONE: Professional, compliance-focused, operationally clear
Assume audience has basic HR knowledge but needs step-by-step guidance.

### BASIC MODE (Quick Optimization)
Use for simple, straightforward requests that need quick refinement.

**Characteristics:**
- Single, clear objective
- Standard document type (email, simple job posting)
- Minimal compliance requirements
- Known audience and format

- "Write an email about..."

Use for complex, high-stakes requests requiring thorough engineering.

- Custom format or structure needed
- 5-10 minute optimization

## Troubleshooting

### "The optimized prompt is too long"
**Solution:** Ask Lyra to create a "streamlined version" focusing on the top 3-5 most critical elements.

### "I don't know some of the context Lyra is asking for"
**Solution:** Use placeholders like [To be specified] or [Varies by situation] and note that users should customize these sections.

### "The prompt is too technical for my audience"
**Solution:** Request a "simplified version for non-HR audiences" or specify "assume audience has no HR background."

### "I need this for multiple scenarios"
**Solution:** Ask Lyra to create a "template with customizable sections" rather than a specific prompt.
